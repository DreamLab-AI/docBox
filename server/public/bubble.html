docBox chat bubble

An embeddable, dependency-free chat widget for the docBox control plane (milestone M7).

M7 puts the primary user's chat surface anywhere a single script tag can go. The widget below is the same asset the control plane serves at /bubble.js; it opens a small panel and relays each message to the box's /api/chat endpoint, rendering the reply inline. There is no framework, no build step, and nothing to install — it is one file.

To embed it on another page, drop this one line where you want the bubble to appear:

<script src="http://YOUR-BOX/bubble.js" data-docbox-url="http://YOUR-BOX"></script>

The data-docbox-url attribute points the widget at your control plane. Omit it and the widget calls the same origin that served the script. Replies come from the box's engine — the deterministic mock by default, or the live engine when the box runs with DOCBOX_ENGINE=live. An external dashboard embed (a different origin) needs the box started with DOCBOX_ALLOWED_ORIGIN set so the browser permits the request.

The live bubble is in the bottom-right corner of this page. It is loaded exactly as an embed would be — <script src="bubble.js"></script>, resolved same-origin.
